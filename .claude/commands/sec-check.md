以下のチェックリストに基づいて、$ARGUMENTS（未指定の場合は `git diff HEAD` の変更差分）をセキュリティレビューしてください。

## 1. XSS / テンプレートインジェクション
- `v-html` にユーザー入力・外部データが直接渡されていないか
- `DOMPurify.sanitize()` を通しているか
- `eval()` / `innerHTML` の直接使用がないか
- URL に `javascript:` スキームが使われていないか

## 2. 認証・認可
- すべての API リクエストに認証チェックがあるか
- JWT 検証がサーバーサイド（Lambda）で行われているか
- IDOR（Insecure Direct Object Reference）の可能性がないか（他ユーザーのリソースにアクセスできないか）
- 認証不要なエンドポイントに `// PUBLIC:` コメントがあるか

## 3. シークレット・機密情報
- APIキー・パスワード・トークンがハードコードされていないか
- `VITE_` 環境変数にシークレットが含まれていないか
- `console.log` に個人情報・認証情報が含まれていないか
- エラーメッセージに内部情報（スタックトレース・スキーマ情報）が含まれていないか

## 4. TypeScript 型安全
- `any` 型が使われていないか
- `as unknown as T` のような強制キャストがないか
- 外部データ（API レスポンス・URL パラメータ）に zod 等で型バリデーションがあるか
- null / undefined の安全なハンドリングがされているか

## 5. 依存関係
- 新規追加パッケージに既知の脆弱性がないか（`npm audit` で確認）
- 削除すべき unused なパッケージが含まれていないか

## 6. AWS / インフラ（IaC ファイルが含まれる場合）
- S3 バケットポリシーが最小権限か（パブリックアクセスブロック有効か）
- IAM ロール・ポリシーが最小権限の原則に従っているか
- CloudFront に HTTPS 強制が設定されているか
- Lambda 関数に不要な権限が付与されていないか
